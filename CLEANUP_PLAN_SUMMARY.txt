# TwinStar-Quantum 프로젝트 구조 정리 계획서

## 1. 권장 모듈 트리 (Proposed Module Tree)
프로젝트의 유지보수성과 확장성을 위해 다음과 같은 구조를 제안합니다.

```
TwinStar-Quantum/
├── core/                # 핵심 로직 (전략, 최적화, 백테스트 엔진)
│   ├── strategy_core.py
│   ├── optimizer.py
│   └── multi_symbol_backtest.py
├── ui/                  # PyQt6 기반 UI (Legacy GUI/ 통합)
│   ├── widgets/
│   ├── design_system/
│   └── trading_dashboard.py
├── exchanges/          # 거래소 API 연동 모듈들
├── utils/              # 공통 유틸리티 (메트릭, 로깅, 데이터 매니저)
├── data/               # SQLite DB, 캐시(Parquet), 설정 파일 등
├── tests/              # 단위 테스트 및 검증 스크립트 (정리된 상태 유지)
├── docs/               # 설계 문서 및 매뉴얼
├── scripts/            # 유틸리티 스크립트 (루트의 일회성 .py 이동)
├── run_gui.py          # 메인 엔트리 스크립트
├── requirements.txt    # 의존성 목록
└── README.md           # 프로젝트 개요
```

## 3. 향후 작업 권장 사항
- **GUI/ 제거**: `GUI/` 디렉토리의 잔여물을 모두 `ui/`로 이전 완료 후 삭제.
- **scripts/ 활용**: 루트에 산재한 독립 실행 스크립트들을 `scripts/` 폴더로 모아 관리.
- **자동화**: 정기적으로 `logs/` 및 `temp/`를 비우는 cleanup 스크립트 유지보수.
