üîê API Related Errors

Binance -2015 Error

"Invalid API-key, IP, or permissions for action"

Cause

API permission or IP configuration issue

Solution

  1. Go to Binance API Management
  2. Verify Enable Futures permission is active
  3. Check IP Access Restrictions:
    • Select Unrestricted, or
    • Register your exact IP address
  4. Confirm Testnet vs Production key
  5. Generate a new API key and try again

OKX/Bitget Authentication Failed

"Authentication failed" or "Invalid passphrase"

Cause

Missing or incorrect Passphrase

Solution

  1. Enter the exact Passphrase you created during API setup
  2. Check case sensitivity
  3. Verify the Passphrase field exists in Settings tab
  4. If Passphrase is lost ‚Üí Regenerate API key

Korean Exchange Balance Shows 0

"Balance: 0 KRW"

Cause

No KRW balance or spot-only exchange

Solution

  1. Verify you have KRW balance on the exchange
  2. Upbit/Bithumb support spot trading only (no futures)
  3. For futures trading, use Bybit, Binance, OKX, etc.
Leverage does not work on Upbit/Bithumb.

üìä Data Related Errors

parquet not found

"15m parquet not found" or "No data available"

Cause

Data for the symbol has not been downloaded

Solution

  1. Go to the Data tab
  2. Select exchange and symbol
  3. Click Download button
  4. Wait for download to complete

[Crypto] Repeated Messages

"[Crypto] API key decryption complete" appears multiple times

Cause

Redundant loading (does not affect functionality)

Solution

  1. Restart the program
  2. If it persists, will be fixed in next version

ü§ñ Bot Related Issues

No Entry Happening

"[STATUS] No Position | Pending=0"

This is NOT an error!

The bot is waiting because entry conditions are not met.

How to Verify

  1. Check the [PREDICT] section in Bot Log
  2. Review Pattern / RSI / MTF conditions
  3. If conditions are not met, waiting is normal behavior
Not trading when conditions aren't right is correct behavior.

[PREDICT] Log Not Showing

Entry prediction log is not displayed

Cause

Insufficient historical data

Solution

  1. Download data from the Data tab
  2. Stop and restart the bot

Stop Loss Not Working

Stop loss is not being executed

Possible Causes

  • Exchange API delay
  • Rapid price movement (slippage)
  • Network issues

Solution

  1. Check positions directly on the exchange
  2. Close manually if necessary
  3. Verify network connection
If the program closes, SL may not work. Keep the program running while the bot is active.

üíª Program Issues

Program Won't Start

Solutions

  1. Add exception in Windows Defender or antivirus
  2. Run as Administrator
  3. Check folder permissions (C:\TwinStar)
  4. Reinstall the program

Display Issues / Text Not Visible

Solutions

  1. Check Windows display scaling is at 100%
  2. Set DPI scaling override in program compatibility settings
  3. Update graphics drivers

‚ùì Frequently Asked Questions

Q: Does the bot keep running if I close the program?

A: No. Closing the program stops the bot. Open positions remain on the exchange and must be managed manually.

Q: Can I trade multiple symbols at once?

A: Yes. You can run multiple bots simultaneously. However, the maximum number of positions may be limited by your license.

Q: Why are backtest and live results different?

A: Slippage, fees, and real-time price fluctuations cause differences. This is common in all trading systems.

Q: What if my API key is compromised?

A: Immediately delete the API key on the exchange and generate a new one. If withdrawal permission was disabled, asset loss risk is low.
