🔐 API 관련 오류

Binance -2015 에러

"Invalid API-key, IP, or permissions for action"

원인

API 권한 또는 IP 설정 문제

해결 방법

  1. Binance API 관리 페이지 접속
  2. Enable Futures 권한이 활성화되어 있는지 확인
  3. IP Access Restrictions 확인:
    • Unrestricted 선택, 또는
    • 본인 IP 정확히 등록
  4. Testnet 키와 Production 키 구분 확인
  5. API 키를 새로 발급받아 다시 시도

OKX/Bitget 인증 실패

"Authentication failed" 또는 "Invalid passphrase"

원인

Passphrase 누락 또는 오타

해결 방법

  1. API 생성 시 입력한 Passphrase를 정확히 입력
  2. 대소문자 구분 확인
  3. Settings 탭에서 Passphrase 필드가 있는지 확인
  4. Passphrase 분실 시 → API 키 재발급

한국 거래소 잔고 0

"Balance: 0 KRW"

원인

KRW 잔고 없음 또는 현물 전용 거래소

해결 방법

  1. 거래소에 KRW 잔고가 있는지 확인
  2. 업비트/빗썸은 현물만 지원 (선물 거래 불가)
  3. 선물 거래는 Bybit, Binance, OKX 등 이용
업비트/빗썸에서는 레버리지가 작동하지 않습니다.

📊 데이터 관련 오류

parquet not found

"15m parquet not found" 또는 "No data available"

원인

해당 심볼의 데이터가 다운로드되지 않음

해결 방법

  1. Data 탭 클릭
  2. 거래소와 심볼 선택
  3. Download 버튼 클릭
  4. 다운로드 완료까지 대기

[Crypto] 반복 출력

"[Crypto] API 키 복호화 로드 완료" 메시지가 여러 번 출력됨

원인

불필요한 반복 로드 (기능에는 영향 없음)

해결 방법

  1. 프로그램 재시작
  2. 지속될 경우 다음 버전에서 수정 예정

🤖 봇 관련 문제

진입이 안 됨

"[STATUS] No Position | Pending=0"

이것은 오류가 아닙니다!

진입 조건이 충족되지 않아 대기 중인 상태입니다.

확인 방법

  1. Bot Log에서 [PREDICT] 섹션 확인
  2. 패턴 / RSI / MTF 조건 확인
  3. 조건이 미충족이면 대기가 정상입니다
시장 조건에 맞지 않으면 거래하지 않는 것이 올바른 동작입니다.

[PREDICT] 로그가 안 나옴

진입 예측 로그가 표시되지 않음

원인

히스토리 데이터 부족

해결 방법

  1. Data 탭에서 데이터 다운로드
  2. 봇 정지 후 재시작

SL이 작동 안 함

스탑로스가 체결되지 않음

원인

  • 거래소 API 지연
  • 급격한 가격 변동 (슬리피지)
  • 네트워크 문제

해결 방법

  1. 거래소에서 직접 포지션 확인
  2. 필요시 수동으로 청산
  3. 네트워크 연결 상태 확인
프로그램이 종료되면 SL이 작동하지 않을 수 있습니다. 봇 실행 중에는 프로그램을 닫지 마세요.

💻 프로그램 오류

프로그램이 실행되지 않음

해결 방법

  1. Windows Defender 또는 백신 프로그램에서 예외 추가
  2. 관리자 권한으로 실행
  3. 설치 폴더 권한 확인 (C:\TwinStar)
  4. 재설치

화면이 깨짐 / 글자가 안 보임

해결 방법

  1. Windows 디스플레이 설정에서 배율 100% 확인
  2. 프로그램 호환성 설정에서 DPI 스케일링 무시
  3. 그래픽 드라이버 업데이트

❓ 자주 묻는 질문

Q: 프로그램을 종료해도 봇이 계속 작동하나요?

A: 아니요. 프로그램을 종료하면 봇도 함께 종료됩니다. 열린 포지션은 거래소에 남아있으므로 수동으로 관리해야 합니다.

Q: 여러 심볼을 동시에 거래할 수 있나요?

A: 네. 여러 봇을 동시에 실행할 수 있습니다. 단, 라이선스에 따라 최대 포지션 수가 제한될 수 있습니다.

Q: 백테스트와 실매매 결과가 다른 이유는?

A: 슬리피지, 수수료, 실시간 가격 변동 등으로 인해 차이가 발생할 수 있습니다. 이는 모든 트레이딩 시스템에서 공통적인 현상입니다.

Q: API 키가 유출되면 어떻게 하나요?

A: 즉시 거래소에서 해당 API 키를 삭제하고 새로 발급받으세요. 출금 권한이 없다면 자산 손실 위험은 낮습니다.
