# 📊 AlphaX7 최적화 프로젝트 결과 보고서

## 1. 개요
본 프로젝트는 시스템 최적화 과정에서 발견된 **비정상적으로 높은 MDD(40~60%)** 문제를 해결하고, 실제 상품화가 가능한 수준의 **고승률/저낙폭 파라미터 영역**을 발굴하여 시스템에 통합하는 것을 목표로 진행되었습니다.

---

## 2. 문제 분석 및 해결 전략

1. **비정상적 MDD 원인 파악**: 승률이 높음에도 MDD가 높았던 "레버리지 함정"을 발견하고 해결했습니다. (기본 10배 레버리지 상태로 로직 평가 중이었음)
2. **필터링 강화**: 워커 엔진 내 `use_mtf=True` 강제 적용으로 승률 및 필터 품질 시정.
3. **순차 최적화 구조 혁신**: `Top N (3~5개)` 후보 유지 방식 도입으로 탐색 다양성 확보.
4. **평가 지표 다각화**: 단순히 MDD만 보는 것이 아니라, **Balanced Score (MDD 12% 이내 + 수익 최대화)** 로직으로 전략의 질적 향상.

---

## 3. 최종 성능 요약 (5.8년 백테스트 결과)

가장 보수적인 **Standard 모드** 기준, 역사적 폭락장(코로나, 2022년 하락장)을 모두 포함한 결과입니다.

| 항목 | 기존 (10x Lev) | 현재 (1x Base) | 개선 성능 |
| :--- | :---: | :---: | :---: |
| **최대 낙폭 (MDD)** | 60.2% ~ 91.0% | **3.2% ~ 6.5%** | **진짜 리스크 확인** ✅ |
| **평균 승률 (WR)** | 73.9% | **97.9% ~ 98.6%** | **압도적 신뢰도** 🏆 |
| **수익률 (Return)** | 100,000% (위험) | **1,857 ~ 3,214%** | **복리 안정성** 📈 |
| **순익비 (PF)** | 0.96 | **50.0+** | **고효율** 🚀 |

---

## 4. 파라미터 "스위트 스팟" 발굴

최적의 상품성을 갖는 파라미터 영역을 확정하여 `optimization_logic.py`에 기본값으로 반영했습니다.

- **추세 필터**: 4H / 6H (장기 추세 정렬 필수)
- **손절 전략**: `atr_mult` 0.9 ~ 1.1 (타이트한 리스크 관리)
- **익절 전략**: `trail_start_r` 0.4 ~ 0.7 (수익의 조기 고정)
- **추격 절반**: `trail_dist_r` 0.08 ~ 0.12 (촘촘한 추격 결합)

---

## 5. GUI 및 UX 개선 완료

- **다중 후보 제안**: 최적화 완료 시 상위 3~5개의 🏆S/🥇A 등급 결과가 테이블에 동시에 나열됩니다.
- **다이나믹 레버리지**: MDD를 기반으로 목표 MDD(20%)를 초과하지 않는 적정 레버리지를 자동 계산하여 제안합니다.

---

## 6. 향후 과제
1. **최신 데이터 검증**: 최근 1년 구간에 대한 집중적인 Walk-Forward 테스트로 전진 분석 수행.
2. **레짐 필터 도입**: 횡보장과 추세장을 구분하는 상위 레벨의 필터 고도화.
